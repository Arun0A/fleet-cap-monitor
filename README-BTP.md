# O2C CAP project — SAP BTP deployment notes

This file contains quick instructions to deploy the O2C CAP app to SAP BTP (Cloud Foundry).

Prerequisites
- An SAP BTP account (trial or paid) with Cloud Foundry enabled
- `cf` CLI installed and logged in (`cf login -a <API> -u <user> -o <org> -s <space>`)
- `cf` service instances created for HANA, XSUAA and Event Mesh (or use equivalent names)
- Node.js toolchain for local testing (prefer inside `nix develop` created for this repo)

Quick local dev
1. Enter the dev shell from repository root:
```zsh
nix develop
cd o2c-cap-project
npm install
npm run watch
```

Build & deploy to Cloud Foundry
1. Build the CAP project for deployment:
```zsh
cd o2c-cap-project
npm run build   # runs `cds build` -> produces `gen/` directory
```
2. Create or bind required service instances in your CF space (`o2c-hana`, `o2c-xsuaa`, `o2c-eventmesh` are example names). Use the BTP cockpit or `cf create-service`.
3. Push the app:
```zsh
cf push -f manifest.yml
```

Notes
- `xs-security.json` contains example roles and scopes. Create an XSUAA instance and bind it to the app; adapt `xs-security.json` if you want different role names.
- `manifest.yml` references `gen/srv` as the app path generated by `cds build`. If you choose a different packaging approach adjust the `path` accordingly.
- Event Mesh is optional for the MVP; if not used remove it from `manifest.yml` and the service bindings.

Events / Event Mesh
- The project now publishes an `OrderConfirmed` event when an order is confirmed. The behavior is:
	- In-process: `srv.emit('OrderConfirmed', payload)` is emitted and handled by an in-process subscriber.
	- External (BTP/Event Mesh): the app will attempt to `cds.connect.to('messaging')` and call `emit/publish` if the messaging service is bound.
	- Local dev fallback: when no external messaging is configured the in-process subscriber still runs so the flow remains functional.

Testing events locally
- By default the dev server allows in-process events. To simulate production enforcement, set `CDS_ENV=cf` when starting.
- To test external messaging in a BTP deployment, bind an Event Mesh instance and ensure the `messaging` connection name is available in `vcap_services` (or service bindings).
